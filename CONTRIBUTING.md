# Contributing

1. Read `AGENTS.md` and `docs/OPERATIONS.md`.
2. Create a focused branch from `main`.
3. Make one coherent change with matching documentation.
4. Run `npm run check` and preview with `npm run dev` when behavior or layout changes.
5. Open a pull request using the repository template.
6. Merge only after validation passes and the release risk is understood.

Do not commit credentials, customer data, generated audit dumps, or tool authentication artifacts.
