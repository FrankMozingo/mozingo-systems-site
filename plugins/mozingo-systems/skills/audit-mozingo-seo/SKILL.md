# Audit Mozingo SEO

1. Read `docs/seo/SEO-PLAYBOOK.md` and `docs/brand/business-identity.md`.
2. Run `npm run check` to establish the repository baseline.
3. Separate findings into crawlability, indexability, relevance, trust, and conversion.
4. Verify live behavior when the request concerns production; do not infer deployment state from the working tree.
5. Use Google Search Console, Ahrefs, SEMrush, or Firecrawl only when connected and useful. Discover current tool schemas before calling external services.
6. Prioritize fixes by business impact, confidence, and effort.
7. Preserve one canonical URL per public page and keep `sitemap.xml` aligned.
8. Do not create doorway pages, keyword stuffing, fabricated local offices, or unsupported claims.
9. Record recurring actions and measurement cadence in `docs/OPERATIONS.md`.

Treat rankings and traffic as observed outcomes, never guarantees.
